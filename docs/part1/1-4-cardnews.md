### [개념] 파이프라인 흐름과 사용할 데이터

오늘 실습의 전체 흐름이에요:

| 단계 | 내용 |
|:--|:--|
| 1. 데이터 | 서울 전체 실거래가 CSV |
| 2. 선별 | 26년 실거래가 중 최고가 + 최저가 각 1건 |
| 3. 디자인 | HTML 템플릿으로 1080x1080 인스타 카드 생성 |
| 4. 결과물 | 카드뉴스 PNG 이미지 완성 |

CSV 파일: `실거래가_단계적매칭_서울전체_완전판_오차10%.csv`

이 파일에는 서울 전체 상업용 부동산 실거래 데이터가 들어 있습니다.

데이터에 담긴 정보:
- 시군구, 지번 주소, 도로명 주소, 건축물 용도, 거래금액, 전용/연면적, 대지면적, 건축년도 등

<details>
<summary>이 데이터가 만들어진 과정 (클릭해서 펼치기)</summary>

국토교통부에서 제공하는 실거래가 데이터에는 한 가지 문제가 있습니다. 개인정보 보호를 위해 지번이 마스킹(***) 처리되어 있어서, 정확히 어떤 건물이 거래된 것인지 알 수 없습니다.

> 예시: 강남구 청담동 *** 번지 → 어떤 건물인지 특정 불가

이 문제를 해결하기 위해, 건축물대장 표제부라는 공적 장부를 활용했습니다. 표제부에는 건물의 정확한 주소와 상세 정보가 기록되어 있는데요, 실거래가 데이터와 표제부 데이터에서 공통으로 존재하는 3가지 정보(연면적, 대지면적, 건축년도)를 기준으로 대조하면 마스킹된 지번과 도로명 주소를 복원할 수 있습니다.

다만, 실제 데이터에서는 면적이 정확히 일치하지 않는 경우도 있습니다. 예를 들어 건물이 코너에 위치하면 실제 사용 가능한 대지면적이 공부상 면적과 다를 수 있고, 옥탑 등 일부 공간이 면적에 포함되지 않는 경우도 있습니다. 그래서 면적에 약간의 오차 범위(±10%)를 허용하는 방식으로 매칭을 진행했습니다.

오늘 활용할 데이터는 위의 전처리가 완료된 일부 데이터입니다.

</details>

> 참고: 본 실습에서 사용하는 CSV 파일은 2026년 2월 23일 기준일자의 데이터입니다.

이 데이터에서 2026년 실거래가 중 최고가와 최저가 각 1건을 뽑아 카드뉴스 샘플을 만들어 볼게요.

오늘 사용할 2건은 다음과 같아요:

| 구분 | 주소 | 거래금액 |
|:--|:--|:--|
| 최고가 | 서울특별시 마포구 양화로 141 (동교동 160-5번지) | 2,650억원 |
| 최저가 | 서울특별시 구로구 도림로10길 23 (구로동 791-58번지) | 1억 6,500만원 |

### [실습] 카드뉴스 생성

클로드 코드에 아래 프롬프트를 입력해 주세요:

[클로드 코드에 입력]
```
"이 폴더에 있는 실거래가 CSV 파일을 읽고,
사전 공유된 templates 폴더의 HTML 템플릿을 참고해서
인스타그램 카드뉴스 PNG 이미지를 만들어줘.

데이터 선별 기준:
- 유형이 "일반"인 것만 (상가/집합 제외)
- 표제부 매칭이 성공한 것만 (지번 주소가 확보된 데이터)
- 2026년 계약 건만
- 최고가 1건 + 최저가 1건 (최저가는 1억 이상)

슬라이드 구성:
- 1장: 표지 (제목 + 날짜)
- 2장: 최고가 거래 데이터 (주소, 거래금액, 면적, 용도, 건축년도)
- 3장: 최저가 거래 데이터

디자인 사양:
- 크기: 1080x1080px
- 배경: 건물 사진 + 그라데이션 오버레이
- 강조색: 개나리 노란색 (#FFD700)
- 텍스트: 흰색, 깔끔한 산세리프
- 데이터 테이블: 우측 하단 배치
- 파일명: 지번 주소로 저장 (예: slide_2_동교동 160-5번지.png)"
```

### [실습] 결과물 확인

커맨드가 정상적으로 실행되면 터미널에 아래와 같은 결과가 출력돼요:

```
선별된 2건:
  최고가: 서울특별시 마포구 동교동 - 2650억
    지번: 서울특별시 마포구 동교동 160-5번지
    도로명: 서울특별시 마포구 양화로 141 (동교동)
    용도: 숙박 | 연면적: 27852.23㎡ | 대지면적: 2277㎡ | 건축년도: 2017.0
  최저가: 서울특별시 구로구 구로동 - 1억 6,500만
    지번: 서울특별시 구로구 구로동 791-58번지
    도로명: 서울특별시 구로구 도림로10길 23 (구로동)
    용도: 제1종근린생활 | 연면적: 21.6㎡ | 대지면적: 21㎡ | 건축년도: 1983.0

slide_1_cover.png 생성 완료
slide_2_동교동 160-5번지.png 생성 완료
slide_3_구로동 791-58번지.png 생성 완료

완료! output 폴더에 총 3장의 PNG가 생성되었습니다.
```

생성된 카드뉴스 PNG는 아래 폴더에 저장돼요:

```
📁 나나_워크숍/output/
├── slide_1_cover.png              ← 표지
├── slide_2_동교동 160-5번지.png    ← 최고가
└── slide_3_구로동 791-58번지.png   ← 최저가
```

결과물을 확인하는 방법은 두 가지가 있어요:

방법 1. 파일 탐색기에서 `나나_워크숍/output` 폴더를 직접 열어 PNG 파일을 더블클릭해 주세요.

방법 2. 클로드 코드에 아래와 같이 요청해 보세요:

[클로드 코드에 입력]
```
"output 폴더의 PNG 파일들을 열어줘"
```

각 슬라이드가 어떻게 나왔는지 확인해 보세요. 마음에 안 드는 부분이 있다면:

[클로드 코드에 입력]
```
"2번 슬라이드에서 거래금액 폰트를 더 크게 해줘"
```

이렇게 대화하면서 수정할 수 있어요. 이것이 클로드 코드의 강점이에요 — 대화로 디자인을 반복 수정할 수 있어요.

### [실습] 자동화 커맨드로 카드뉴스 만들어보기

위에서는 클로드 코드에 상세한 프롬프트를 입력해서 카드뉴스를 만들었어요. 이번에는 이미 만들어진 자동화 스크립트를 활용해서 커맨드 두 줄로 카드뉴스를 생성해 볼게요.

Step 1. 데이터 확인 — 어떤 매물이 선별되는지 미리 확인해 주세요:

```
node render_cards.js --prepare
```

> 최고가와 최저가 매물의 지번, 도로명, 거래금액 등이 표시돼요. 원하는 데이터가 맞는지 확인해 주세요.

Step 2. 카드뉴스 생성 — 확인이 끝나면 아래 명령어로 카드뉴스를 생성해 주세요:

```
node render_cards.js --now
```

> 이 스크립트는 CSV 데이터 읽기 → 필터링 → 선별 → 템플릿 적용 → PNG 생성까지 전 과정을 자동으로 수행해요.

실행이 완료되면 `나나_워크숍/output/` 폴더에 새로운 카드뉴스 PNG가 생성돼요. 결과를 확인해 보세요.

> 참고: 이 스크립트는 앞에서 클로드 코드에게 요청했던 작업을 코드로 자동화한 것이에요. 프롬프트로 한 단계씩 요청하는 것과, 스크립트로 한 번에 실행하는 것의 차이를 느껴 보세요.
