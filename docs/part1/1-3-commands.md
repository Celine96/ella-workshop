### [개념] 슬래시 명령어 한눈에 보기

클로드 코드 안에서 `/`로 시작하는 명령어를 사용할 수 있어요. 전부 외울 필요는 없고, 필요할 때 이 표를 참고하세요.

세션 관리 (가장 많이 씀)

| 명령어 | 하는 일 |
|:--|:--|
| `/clear` | 대화를 초기화합니다. 파일은 그대로 유지됩니다. 대화가 길어져서 새 작업으로 넘어갈 때 사용해요 |
| `/compact` | 대화를 압축해서 토큰을 절약하고, 클로드가 앞부분을 까먹는 걸 방지합니다. 대화가 길어져서 같은 작업을 이어가야 할 때 사용해요 (컨텍스트가 일정 수준 이하로 떨어지면 자동 실행) |
| `/resume` | 클로드 코드를 껐다 켜도 이전 대화를 복원합니다. 퇴근 후 다음날 이어서 작업할 때 유용합니다 |

모델 & 사용량

| 명령어 | 하는 일 |
|:--|:--|
| `/model` | 모델을 전환합니다. Sonnet(빠름) ↔ Opus(똑똑함) |
| `/cost` | 현재 세션의 비용을 확인합니다 |
| `/usage` | 전체 사용량을 확인합니다 |

컨텍스트 & 파일

| 명령어 | 하는 일 |
|:--|:--|
| `/context` | 클로드가 현재 인식하고 있는 파일 목록을 봅니다 |
| `/add-dir` | 참고할 폴더를 추가합니다 |
| `/init` | 새 프로젝트에서 CLAUDE.md를 자동 생성합니다 |
| `/memory` | 프로젝트 기억(메모리)을 확인하거나 수정합니다 |

에이전트 & 권한

| 명령어 | 하는 일 |
|:--|:--|
| `/agents` | 에이전트를 생성하고 관리합니다 |
| `/permissions` | 도구 사용 권한을 설정합니다 |

기타

| 명령어 | 하는 일 |
|:--|:--|
| `/help` | 사용 가능한 모든 명령어를 봅니다 |
| `/config` | 설정을 변경합니다 |
| `/doctor` | 문제가 있을 때 진단합니다 |
| `/quit` 또는 `/exit` | 클로드 코드를 종료합니다 |

### [개념] 3가지 입력 모드

`Shift+Tab`을 누르면 모드가 전환돼요.

| 모드 | 설명 | 언제 쓰나요? |
|:--|:--|:--|
| `accept edits on` | 파일 편집은 자동으로 승인하지만, 셸 명령어 실행은 승인을 구합니다 | 익숙해진 후, 신뢰할 수 있는 작업 |
| `plan mode on` | 클로드가 먼저 계획을 보여주고, 승인하면 실행합니다 | 복잡한 작업을 단계별로 확인하고 싶을 때 |
| `auto mode on` | 파일 편집과 명령어 실행을 모두 자동으로 승인합니다 | 완전 자동화가 필요할 때 |

> Shift+Tab을 처음 누르기 전엔 세 모드 어느 것도 켜지지 않은 기본 상태예요. 처음에는 기본 상태나 `plan mode on`으로 시작하는 걸 권장해요.

### [개념] 키보드 단축키

| 단축키 | 하는 일 |
|:--|:--|
| `Escape` | 클로드가 응답 중일 때 중단시킵니다 |
| `Escape` × 2번 | 되감기 — 대화만 되돌리거나, 파일 변경까지 되돌릴 수 있습니다 |
| `Shift+Tab` | 입력 모드를 순환합니다 (기본 → accept edits on → plan mode on → auto mode on → 기본) |
| `Shift+Enter` | 줄바꿈 (여러 줄 입력할 때) |
| `Ctrl+V` | 이미지를 붙여넣습니다 |

### [개념] Think 키워드

클로드는 기본적으로 즉답하지만, 프롬프트 안에 특정 키워드를 넣으면 답변 전에 더 오래 생각하게 할 수 있어요. 여러 관점을 따져야 하는 결정에 유용해요.

| 키워드 | 언제 쓰나요? | 생각 시간·비용 |
|:--|:--|:--|
| `think` | 간단한 비교·정리. "이 두 문구 중 어느 게 더 끌려?" 같은 가벼운 판단 | 기본보다 약간 증가 |
| `think hard` / `think harder` | 여러 요소를 동시에 고려해야 하는 결정. 예: "이 실거래 데이터에서 어떤 매물 3개 뽑아야 인스타에서 가장 반응 좋을까? 가격대·지역·스토리성 고려해서"<br/>예: "이 PRD 다 썼는데 내가 놓친 부분 있을까? 기획자 관점으로 다시 봐줘"<br/>예: "이 카드뉴스 초안 중에 법률적으로 문제 될 표현 있는지 꼼꼼히 봐줘" | 중간 정도 증가 |
| `ultrathink` | 실수하면 되돌리기 어려운 중요 결정. 예: 브랜드 네이밍 후보 최종 3개 압축, 공식 발표 전 전체 캠페인 리뷰 | 크게 증가 (응답 느려지고 비용 많이 들어감) |

> 참고: 세 단계는 Anthropic 엔지니어링 블로그에서 공개된 패턴이에요. 내부적으로 쓰는 "생각 토큰" 예산이 점점 커진다고 알려져 있지만, 정확한 숫자는 공식 문서에 명시되어 있지 않아요.

사용 원칙:
- "빨리 답 나오면 되는 일"은 키워드 없이
- "여러 관점에서 따져봐야 하는 일"은 `think harder`
- 결과물이 사람들에게 배포되는 중요 결정이면 `ultrathink` 고려

사용법: 프롬프트 안에 자연스럽게 넣으면 돼요.

[클로드 코드에 입력]
```
"이 폴더 구조를 think harder로 분석해서 리팩토링 포인트 3개만 뽑아줘"
```

## 실습 시작 안내

이번 워크숍의 실습은 커스텀 커맨드로 진행해요. 클로드 코드에서 `/`를 입력하면 사용 가능한 커맨드 목록이 나타납니다. 각 실습의 첫 번째 커맨드를 입력하면 안내가 시작되고, Step이 끝나면 다음 커맨드를 안내해 드려요.

---

### [실습 1] 연습 1: 모드 전환 & 자율 판단 체험

> 커맨드: `/1-3-practice1-step1` ~ `/1-3-practice1-step3`

Step 1. `Shift+Tab`을 눌러 "plan mode on"으로 전환해볼게요.

Step 2. 클로드에게 아래를 입력하고, 응답을 관찰해볼게요:

[클로드 코드에 입력]
```
"이 폴더를 분석하고, 여기서 어떤 작업이 가능한지 제안해줘"
```

관찰 포인트:
- 어떤 파일을 읽었는지
- 어떤 제안을 하는지
- 계획을 보여주고 승인을 구하는지 (plan mode on)

Step 3. `Shift+Tab`을 한 번 더 눌러 "auto mode on"으로 전환한 뒤, 같은 요청을 다시 해볼게요. 차이를 느껴 보세요.

---

### [실습 2] 연습 2: 세션 관리 3종 체험

> 커맨드: `/1-3-practice2-step1` ~ `/1-3-practice2-step3`

> 작업이 길어지면 클로드 코드가 컨텍스트 부족을 알려줘요. 이때 `/compact`(이어가기) 또는 `/clear`(초기화)로 해결할 수 있어요.

Step 1. `/compact`를 입력해볼게요 → 대화가 압축되는 것을 확인해 보세요.

Step 2. `/cost`를 입력해볼게요 → 현재까지의 비용을 확인해 보세요.

Step 3. 종료 → 재실행 → 복원을 한 번에 체험해볼게요:
1. `/quit` 또는 `/exit`으로 클로드 코드를 종료해 주세요.
2. 터미널에서 `claude`를 입력해서 다시 실행해 주세요.
3. `/resume`을 입력해 주세요 → 이전 대화가 복원되는 것을 확인할 수 있어요.

> 이 3가지(`/compact`, `/cost`, `/resume`)는 실무에서 매일 쓰게 돼요!
