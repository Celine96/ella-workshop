## 1.3 명령어 & 네비게이션

### [개념] 슬래시 명령어 한눈에 보기

클로드 코드 안에서 `/`로 시작하는 명령어를 사용할 수 있습니다. 전부 외울 필요는 없고, 필요할 때 이 표를 참고하세요.

**세션 관리 (가장 많이 씀)**

| 명령어 | 하는 일 |
|--------|--------|
| `/clear` | 대화를 초기화합니다. 파일은 그대로 유지됩니다 |
| `/compact` | 대화가 길어졌을 때 압축합니다. 토큰을 절약하고 클로드가 앞부분을 까먹는 걸 방지합니다 |
| `/resume` | 클로드 코드를 껐다 켜도 이전 대화를 복원합니다. 퇴근 후 다음날 이어서 작업할 때 유용합니다 |

**모델 & 사용량**

| 명령어 | 하는 일 |
|--------|--------|
| `/model` | 모델을 전환합니다. Sonnet(빠름) ↔ Opus(똑똑함) |
| `/cost` | 현재 세션의 비용을 확인합니다 |
| `/usage` | 전체 사용량을 확인합니다 |

**컨텍스트 & 파일**

| 명령어 | 하는 일 |
|--------|--------|
| `/context` | 클로드가 현재 인식하고 있는 파일 목록을 봅니다 |
| `/add-dir` | 참고할 폴더를 추가합니다 |
| `/init` | 새 프로젝트에서 CLAUDE.md를 자동 생성합니다 |
| `/memory` | 프로젝트 기억(메모리)을 확인하거나 수정합니다 |

**에이전트 & 권한**

| 명령어 | 하는 일 |
|--------|--------|
| `/agents` | 서브에이전트를 생성하고 관리합니다 |
| `/permissions` | 도구 사용 권한을 설정합니다 |

**기타**

| 명령어 | 하는 일 |
|--------|--------|
| `/help` | 사용 가능한 모든 명령어를 봅니다 |
| `/config` | 설정을 변경합니다 |
| `/doctor` | 문제가 있을 때 진단합니다 |
| `/quit` 또는 `/exit` | 클로드 코드를 종료합니다 |

### [개념] 3가지 입력 모드

`Shift+Tab`을 누르면 모드가 전환됩니다.

| 모드 | 설명 | 언제 쓰나요? |
|------|------|-------------|
| **Default (기본)** | 클로드가 작업하기 전에 매번 허락을 구합니다. 읽기 작업만 자동 실행됩니다 | 처음 배울 때 (기본값) |
| **Accept Edits (자동 승인)** | 파일 편집과 일반적인 파일 작업을 자동으로 승인합니다 | 익숙해진 후, 신뢰할 수 있는 작업 |
| **Plan (계획)** | 클로드가 먼저 계획을 보여주고, 승인하면 실행합니다 | 복잡한 작업을 단계별로 확인하고 싶을 때 |

> 처음에는 **Default 모드**(기본값)로 사용하면서 하나씩 확인하는 것을 권장합니다.

### [개념] 키보드 단축키

| 단축키 | 하는 일 |
|--------|--------|
| `Escape` | 클로드가 응답 중일 때 중단시킵니다 |
| `Escape` × 2번 | 되감기 — 대화만 되돌리거나, 파일 변경까지 되돌릴 수 있습니다 |
| `Shift+Tab` | 입력 모드를 전환합니다 (Default → Accept Edits → Plan) |
| `Shift+Enter` | 줄바꿈 (여러 줄 입력할 때) |
| `Ctrl+V` | 이미지를 붙여넣습니다 |

### [개념] Think 키워드

클로드에게 "더 깊이 생각해"라고 요청할 수 있습니다.

| 키워드 | 효과 |
|--------|------|
| `think` | 일반적인 분석 |
| `think harder` | 더 깊은 분석 |
| `ultrathink` | 최대 분석 깊이 (무지개 표시가 뜸). 쿼터를 더 많이 사용합니다 |

사용법: 프롬프트 안에 자연스럽게 넣으면 됩니다.

**[클로드 코드에 입력]**
```
"이 데이터를 think harder로 분석해서 패턴을 찾아줘"
```

### [실습] 연습 1: 모드 전환 & 자율 판단 체험

**Step 1.** `Shift+Tab`을 눌러 현재 모드를 확인합니다. Edit 모드인지 확인하세요.

**Step 2.** 클로드에게 아래를 입력합니다:

**[클로드 코드에 입력]**
```
"이 폴더를 분석하고, 여기서 어떤 작업이 가능한지 제안해줘"
```

**Step 3.** 클로드의 응답을 관찰합니다:
- 어떤 파일을 읽었는지
- 어떤 제안을 하는지
- 파일을 읽기 전에 허락을 구하는지 (Edit 모드)

**Step 4.** `Shift+Tab`으로 Plan 모드로 전환한 뒤, 같은 요청을 다시 해봅니다. 차이를 느껴보세요.

---

### [실습] 연습 2: 세션 관리 3종 체험

**Step 1.** `/compact`를 입력합니다 → 대화가 압축되는 것을 확인합니다.

**Step 2.** `/cost`를 입력합니다 → 현재까지의 비용을 확인합니다.

**Step 3.** 클로드 코드를 종료합니다: `/quit` 또는 `Ctrl+D`

**Step 4.** 다시 실행합니다:
```
claude
```

**Step 5.** `/resume`을 입력합니다 → 이전 대화가 복원되는 것을 확인합니다.

> 이 3가지(`/compact`, `/cost`, `/resume`)는 실무에서 매일 쓰게 됩니다!
