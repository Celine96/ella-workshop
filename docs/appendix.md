## 실무 팁

### CLAUDE.md 활용법

CLAUDE.md는 클로드 코드가 매 세션마다 자동으로 읽는 프로젝트 설정 파일입니다. 프로젝트 폴더에 이 파일을 만들어두면, 매번 같은 맥락을 다시 설명할 필요 없이 클로드가 프로젝트를 기억한 상태로 작업을 시작합니다.

CLAUDE.md에 담으면 좋은 내용:

- 프로젝트 개요와 목적
- PRD (에이전트 구성, 파이프라인, 품질 기준)
- 용어 규칙 (예: "ALWAYS use 워크숍, not 세미나")
- 브랜드 톤앤매너
- 금지 사항 (예: "NEVER use 확정수익, 원금보장")

> CLAUDE.md는 프로젝트의 "헌법"과 같습니다. 개별 프롬프트보다 우선 적용되며, 명시적으로 수정하지 않는 한 계속 유지됩니다.

### 실무에서 바로 쓸 수 있는 프롬프트 4가지

1. UX가 어색할 때

[클로드 코드에 입력]
```
"10년차 UX 디자이너로서 이 제품의 사용자 흐름을 평가해. 문제점을 찾고, 개선안을 우선순위와 근거를 붙여서 제안해."
```

2. 뭘 먼저 고쳐야 할지 모를 때

[클로드 코드에 입력]
```
"이 제품의 PM으로서 아래 고객 피드백과 이탈률 데이터를 분석해. 지금 당장 손봐야 할 항목 3가지를 임팩트 순으로 뽑아."
```

3. 코드가 지저분해졌을 때

[클로드 코드에 입력]
```
"시니어 엔지니어로서 코드베이스를 리뷰해. 성능 병목, 중복, 접근성 이슈를 진단하고 리팩토링 계획을 잡아."
```

4. 출시 전 불안할 때

[클로드 코드에 입력]
```
"QA 리드로서 이 제품을 테스트해. 엣지 케이스, 에러 처리 누락, UI 깨짐을 심각도 순으로 정리해."
```

> 이 프롬프트들의 공통점: 역할을 명확히 지정하고, 구체적인 산출물을 요청합니다. 2.3에서 배운 Generator/Evaluator 분리를 프롬프트 레벨에서 활용하는 것이에요.

## 워크숍 후 다음 단계

1. 만든 에이전트를 매일 사용해보기 — 실사용하면서 개선점 찾기
2. CLAUDE.md 계속 업데이트하기 — 새 규칙이나 패턴 발견 시 추가
3. 파이프라인 확장하기 — 카드뉴스 → 자동 게시 → 성과 분석까지 연결
