## 2.5 실무 팁

---

워크숍을 마치고 실무에서 바로 활용할 수 있는 팁을 정리했어요. CLAUDE.md 활용법과 실무에서 효과가 검증된 프롬프트 4가지를 소개합니다.

---

### CLAUDE.md 활용법

CLAUDE.md는 클로드 코드가 매 세션마다 자동으로 읽는 프로젝트 설정 파일입니다.

CLAUDE.md 없이:
> "나는 부동산 마케팅 기획자야. 카드뉴스를 만들건데, 유형은 일반만, 금지어는 확정수익/원금보장, 이미지는 1080x1080으로..."
> → 매번 새 대화마다 같은 설명을 반복해야 합니다.

CLAUDE.md 있으면:
> "카드뉴스 만들어줘"
> → 클로드가 이미 프로젝트 맥락을 알고 있어서 바로 작업을 시작합니다.

CLAUDE.md에 담으면 좋은 내용:

- 프로젝트 개요와 목적
- PRD (에이전트 구성, 파이프라인, 품질 기준)
- 용어 규칙 (예: "ALWAYS use 워크숍, not 세미나")
- 브랜드 톤앤매너
- 금지 사항 (예: "NEVER use 확정수익, 원금보장")

> CLAUDE.md는 프로젝트의 "헌법"과 같습니다. 개별 프롬프트보다 우선 적용되며, 명시적으로 수정하지 않는 한 계속 유지됩니다.

#### CLAUDE.md를 프롬프트로 만들기

CLAUDE.md를 직접 작성할 필요 없어요. 클로드 코드와 작업하다가 아래 한 줄이면 됩니다.

[클로드 코드에 입력]
```
"지금까지의 내용을 CLAUDE.md에 저장해줘"
```

---

### 실무에서 바로 쓸 수 있는 프롬프트 4가지

1. UX가 어색할 때

[클로드 코드에 입력]
```
"10년차 UX 디자이너로서 이 제품의 사용자 흐름을 평가해. 문제점을 찾고, 개선안을 우선순위와 근거를 붙여서 제안해."
```

2. 뭘 먼저 고쳐야 할지 모를 때

[클로드 코드에 입력]
```
"이 제품의 PM으로서 아래 고객 피드백과 이탈률 데이터를 분석해. 지금 당장 손봐야 할 항목 3가지를 임팩트 순으로 뽑아."
```

3. 코드가 지저분해졌을 때

[클로드 코드에 입력]
```
"시니어 엔지니어로서 코드베이스를 리뷰해. 성능 병목, 중복, 접근성 이슈를 진단하고 리팩토링 계획을 잡아."
```

4. 출시 전 불안할 때

[클로드 코드에 입력]
```
"QA 리드로서 이 제품을 테스트해. 엣지 케이스, 에러 처리 누락, UI 깨짐을 심각도 순으로 정리해."
```

> 이 프롬프트들의 공통점: 역할을 명확히 지정하고, 구체적인 산출물을 요청합니다. 2.3에서 배운 Generator/Evaluator 분리를 프롬프트 레벨에서 활용하는 것입니다.
