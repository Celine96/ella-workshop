## 2.2 PRD (제품 요구사항 문서)

### [개념] PRD(Product Requirements Document)란?

2.1에서 에이전트의 개념을 배웠습니다. 그런데 한 가지 빠진 것이 있습니다. 에이전트에게 "무엇을 해야 하는지"를 어떻게 정확하게 전달할 것인가? 라는 문제입니다.

AI에게 "알아서 카드뉴스 만들어줘"라고 하면 어떻게 될까요? 어떤 데이터를 쓸지, 어떤 디자인으로 만들지, 어떤 기준으로 품질을 판단할지 모두 AI가 임의로 결정하게 됩니다. 결과물이 매번 달라지고, 원하는 품질을 안정적으로 얻기 어렵습니다.

이 문제를 해결하는 것이 바로 PRD(Product Requirements Document)입니다. PRD는 AI 에이전트에게 주는 상세 업무 지시서로, 다음과 같은 핵심 항목들을 담고 있습니다:

| 항목 | 역할 | 예시 |
|------|------|------|
| 목적 | 이 시스템이 왜 필요한지 | "마케팅 콘텐츠를 자동 생성하여 인력 비용을 절감한다" |
| 목표 지표 | 성공을 어떻게 측정할지 | "월간 신규 관심 유저 수" |
| 에이전트 구성 | 누가 어떤 역할을 맡는지 | "기획자, 실행자, 검수자로 분리" |
| 파이프라인 | 작업이 어떤 순서로 흐르는지 | "데이터 수집 → 선별 → 디자인 → 검수 → 게시" |
| 입출력 정의 | 무엇을 넣고 무엇이 나오는지 | "CSV 데이터 → 1080x1080 카드뉴스 PNG" |
| 품질 기준 | 어떤 기준을 지켜야 하는지 | "금지어 미포함, 면책 문구 필수, 이미지 사양 준수" |

PRD가 잘 작성되어 있으면, AI 에이전트는 매번 일관된 품질의 결과물을 만들어낼 수 있습니다.

### [개념] 엘라의 PRD 살펴보기 — 1부에서 체험한 작업의 뒷면

1부(1.4)에서 직접 체험한 '부동산 실거래가 카드뉴스 디자인 파이프라인' 기억하시나요? 실거래가 CSV 데이터를 넣으면 카드뉴스 PNG가 나오는 그 작업입니다.

그 작업이 안정적으로 돌아가려면 뒤에서 PRD가 모든 것을 정의하고 있어야 합니다. 엘라의 PRD 핵심 구조를 살펴볼게요.

| 항목 | 내용 |
|------|------|
| 1. 개요 | 목적: 부동산 마케팅 콘텐츠 자동 생성 & 게시 / 핵심 가치: 수작업 3~4시간/건 → 자동 15분/건 |
| 2. 전략 및 목표 | North Star Metric: 월간 신규 적격 관심 유저 수 / Input Metrics: 주 3회 발행, 게시물당 도달 500, 프로필 방문 5%, 행동 전환 20% |
| 3. 에이전트 팀 | Max(매니저): 일일 계획, KPI 추적 / Aria(콘텐츠): 카드뉴스 생성, 캡션 작성 / Luna(검수): 품질 검사, 법률 준수 |
| 4. 파이프라인 | 데이터 수집 → 선별 → 디자인 → 검수 → 게시 |
| 5. 품질 기준 | 금지어 목록, 면책 문구, 이미지 사양 |

PRD 안에 에이전트별 상세 스펙도 포함할 수 있습니다. 예를 들어 엘라의 PRD에서는 각 에이전트의 사용 도구, 입출력, 판단 기준까지 정의하고 있습니다.

- Aria (콘텐츠): 실거래가 데이터(CSV)를 입력받아 카드뉴스 이미지와 인스타 캡션을 생성합니다. 금지어 미포함, 면책 문구 포함, 1080x1080 사이즈를 품질 기준으로 따릅니다.
- Luna (검수): Aria의 결과물을 입력받아 승인 또는 반려를 판정합니다. 80점 이상이면 자동 승인, 50~79점이면 사람 검토, 50점 미만이면 반려합니다.

### Claude에게 PRD를 요청하는 방법

PRD를 처음부터 직접 작성할 필요는 없어요. 클로드 코드에게 목적과 현재 상황을 간단히 설명하면, AI-Native 방식으로 함께 작성할 수 있습니다.

[클로드 코드에 입력]
```
"나는 [자동화하고 싶은 업무]를 하고 있어.
현재는 [현재 작업 방식]으로 진행하고 있는데,
이걸 자동화하기 위한 PRD를 작성해줘."
```

> 구체적인 PRD 작성 실습은 2.4에서 직접 진행할 예정이에요. 위 예시와 엘라의 PRD를 참고하면서 함께 만들어 볼게요.
